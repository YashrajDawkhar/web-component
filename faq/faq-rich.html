All endpoints require a bearer token in the Authorization header. Generate one in Settings → API keys.

curl https://api.example.com/v1/orders \
  -H "Authorization: Bearer sk_live_••••••" \
  -H "Content-Type: application/json"
Never expose secret keys in client-side code. Use restricted publishable keys for browser apps.

If you hit the limit, you'll get a 429 Too Many Requests response. Back off using the Retry-After header.

Webhooks are POST requests to your endpoint. Verify the signature, then handle the event:

  1. Read the X-Webhook-Signature header
  2. Compute HMAC-SHA256 of the raw body using your signing secret
  3. Compare in constant time
  4. Return 200 within 5 seconds
Pro tip: respond 200 immediately, then process the event in a background job. Slow handlers get retried.
